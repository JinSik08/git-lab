Respuestas a las preguntas

  1. ¿Qué es el DOM?

    El DOM (Document Object Model) es una representación estructurada del documento HTML. Permite a los lenguajes de programación, como JavaScript, acceder y modificar el contenido, estructura y estilo de la página web.

    Fuente: MDN Web Docs

  2. ¿Qué es un nodo?

    Un nodo es una unidad dentro del DOM. Puede ser un elemento (como <p>), un texto, un atributo o un comentario.

  3. ¿Cómo se accede a un elemento del DOM?

    Se puede acceder usando métodos como document.getElementById(), document.querySelector(), entre otros.

    const titulo = document.getElementById("titulo");

  4. ¿Cómo se modifica un elemento del DOM?

    Se puede modificar cambiando propiedades como innerText, innerHTML, classList, style, etc.

    titulo.innerText = "Nuevo título";

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    getElementBy* retorna colecciones o elementos por ID, clase o nombre, mientras que querySelector* acepta selectores CSS y retorna el primer elemento que coincida (o todos si se usa querySelectorAll).

    document.getElementById("id") vs document.querySelector("#id")

  6. ¿Qué es un evento?

    Un evento es una acción del usuario o del navegador que puede ser detectada, como hacer clic, mover el mouse, presionar una tecla, etc.

  7. ¿Qué es un listener?

    Un listener (escuchador) es una función que espera a que ocurra un evento para ejecutarse. Se registra con addEventListener.

    boton.addEventListener("click", () => { alert("Clic!"); });

  8. ¿Qué es un callback?

    Un callback es una función que se pasa como argumento a otra función para ser ejecutada posteriormente, por ejemplo, cuando ocurre un evento.

  9. ¿Qué es un objeto de evento?

    Es un objeto que contiene información sobre el evento que se disparó, como el tipo de evento, el elemento que lo disparó, coordenadas del mouse, etc.

    element.addEventListener("click", function(event) { console.log(event.target); });

Conclusiones

Experiencia general

Usar ChatGPT y GitHub Copilot para esta actividad fue útil y eficiente. ChatGPT me ayudó a comprender los conceptos teóricos del DOM y los eventos, además de generar ejemplos claros y bien comentados. Copilot me ayudó escribiendo fragmentos de código rápidamente mientras programaba en el archivo JavaScript.

En el ejercicio 6, la imagen dejó de mostrarse al aplicar el script generado por ChatGPT. Para depurar el problema, también pedí que el script mostrara los valores de width y height debajo de la imagen para revisar si los datos estaban siendo generados correctamente. Además, con la ayuda de un compañero, cambié la liga original de placekitten.com a placecats.com para que el código funcionara correctamente y las imágenes pudieran mostrarse sin error.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Usaría ChatGPT cuando necesite explicaciones, resolver dudas teóricas, entender errores o generar fragmentos completos de código a partir de una descripción. Usaría Copilot cuando esté escribiendo código en el editor y necesite asistencia rápida, autocompletado inteligente o sugerencias dentro del contexto del archivo actual.

Ventajas de usar AI para programar

  • Ahorra tiempo generando código repetitivo.
  • Ofrece explicaciones claras de conceptos difíciles.
  • Ayuda a aprender mientras se programa.
  • Permite experimentar con nuevas soluciones rápidamente.

Desventajas de usar AI para programar

  • Puede generar código incorrecto o inseguro si no se revisa.
  • Puede fomentar dependencia si no se entiende lo que se escribe.
  • Copilot a veces sugiere cosas fuera de contexto.
