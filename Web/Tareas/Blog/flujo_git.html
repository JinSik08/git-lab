Flujo de trabajo de git y github

Git y GitHub se utilizan para el control de versiones y la colaboración en proyectos de software.

Características del git y github

Git

Git es un sistema de control de versiones distribuido. Esto significa que permite a los desarrolladores rastrear cambios en el código, colaborar con otros y revertir modificaciones si es necesario.

Funciones principales:

  1. Control de versiones: Puedes volver a versiones anteriores del código si algo sale mal.
  2. Trabajo en paralelo: Varios desarrolladores pueden trabajar en el mismo proyecto sin sobrescribirse.
  3. Branching y merging: Puedes crear "ramas" de desarrollo separadas y luego unirlas al código principal.
  4. Distribuido: Cada persona que clona un repositorio tiene una copia completa del historial del proyecto.

GitHub

GitHub es una plataforma en la nube que permite alojar repositorios Git, facilitando la colaboración y gestión de código en equipo.

Funciones principales:

  1. Repositorio remoto: Guarda tu código en la nube y accede desde cualquier lugar.
  2. Colaboración: Puedes trabajar con otros mediante pull requests y code reviews.
  3. Automatización: Con GitHub Actions, puedes automatizar tareas como pruebas y despliegue.
  4. Gestión de proyectos: Usa issues y proyectos para organizar tareas y errores.

Códigos que puede usar:

Códigos de git
Código Definición
git clone "url de SSH" Hacer el clone del repositorio
git status Revisar el estatus del archivo
git add Agregar los cambios al staging area
git commit -m "Mensaje describiendo los cambios" Confirmar los cambios en el historial de Git
git push origin "nombre de rama" Enviar los cambios al repositorio remoto en GitHub
git pull origin "nombre de rama" Actualizar el código desde GitHub
git fetch Descargar los cambios sin fusionarlos automáticamente
git branch Listar las ramas existentes
git checkout "nombre de rama" Cambiar de rama
git merge "nombre de rama a fusionar" Fusionar una rama con otra
